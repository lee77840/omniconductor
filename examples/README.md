# `examples/`

Worked example installs that show **what CONDUCTOR actually writes into a project**.

## `sample-claude-project/` — real Claude Code install

A committed, browsable snapshot of what the Claude adapter produces. It was generated by running:

```bash
bash adapters/claude/transform.sh examples/sample-claude-project \
  --no-prompt \
  --recipes=coding-conventions,tdd,debugging
```

So you can read the rendered rules, agents, hooks, settings, and doc templates without running anything yourself.

### What's inside

```
sample-claude-project/
├── CLAUDE.md                         # orchestrator manual (project entry point)
├── .claude/
│   ├── rules/                        # 5 universal rules + 3 recipes (coding-conventions, tdd, debugging)
│   ├── agents/                       # 6 role definitions (planner, builder, reviewer, helper, designer, scribe)
│   ├── hooks/                        # 7 enforcement hooks (PreToolUse + Stop)
│   ├── hookify.*.local.md            # opt-in hookify rule templates
│   └── settings.json                 # permissions allowlist + hooks registry
└── docs/
    ├── CURRENT_WORK.md               # session-continuity template
    ├── REMAINING_TASKS.md
    ├── PLANS.md / TASKS.md / INDEX.md
    └── specs/_example.md             # spec-as-you-go example
```

### How to regenerate

From the repo root:

```bash
rm -rf examples/sample-claude-project
bash adapters/claude/transform.sh examples/sample-claude-project \
  --no-prompt \
  --recipes=coding-conventions,tdd,debugging
# The committed sample does NOT include the generated .conductor-manifest.json
# (it records absolute host paths and is not portable). Remove it after regenerating:
rm -f examples/sample-claude-project/.conductor-manifest.json
```

### Notes / caveats

- **`.conductor-manifest.json` is intentionally excluded.** The adapter writes it on every
  install to track files for `--uninstall`, but it embeds the absolute install path of the
  machine that ran it, so it is not suitable to commit. A real install will have one.
- **Hooks reference the project root via `$CLAUDE_PROJECT_DIR`**, not absolute paths — the
  committed sample's `settings.json` and hooks are host-agnostic.
- **This snapshot can drift.** It is illustrative, not a fixture the CI diffs against. If the
  adapter, universal rules, or recipes change, the rendered output here may lag until someone
  regenerates it with the command above. Treat it as a "what you get" preview, not the source of truth.

## Other tools

The same idea applies to the other 5 adapters (`cursor`, `copilot`, `gemini`, `codex`, `windsurf`).
To preview any of them into a throwaway dir:

```bash
bash adapters/<tool>/transform.sh /tmp/preview-<tool> --no-prompt --recipes=tdd,debugging
```

## Contributing examples

Community-contributed example installs are welcome — see `docs/CONTRIBUTING.md`. Each example should:

- Be a runnable, minimal project (not a fork of something large).
- Include a `README.md` explaining what the example demonstrates.
- Include the `init` (or `transform.sh`) command used to install CONDUCTOR into it.
- NOT include any secrets, API keys, production data, or absolute host paths.
